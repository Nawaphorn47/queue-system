📚 Queue System API Documentation

เอกสารคู่มือการใช้งาน API สำหรับระบบจัดการคิว (Queue System)

Base URL: http://localhost (หรือ http://localhost:3000 หากไม่ได้ผ่าน Nginx)
POST /reserve จองคิวใหม่

📝 รายละเอียด

ใช้สำหรับให้ลูกค้ากดรับบัตรคิว ระบบจะรันหมายเลขคิวให้อัตโนมัติ

📦 Request Body (JSON)

{
  "name": "ชื่อลูกค้า (String)"
}

✅ Example Response (200 OK)

{
  "message": "จองคิวสำเร็จ!",
  "data": {
    "id": 1,
    "customer_name": "สมชาย",
    "queue_number": 1,
    "status": "waiting",
    "counter_number": null,
    "created_at": "2023-10-25T10:00:00.000Z"
  }
}
PATCH /call-next เรียกคิวถัดไป

📝 รายละเอียด

เจ้าหน้าที่กดปุ่มเพื่อเรียกคิวถัดไปที่รออยู่ (สถานะ waiting -> called) โดยต้องส่งหมายเลขช่องบริการมาด้วย

📦 Request Body (JSON)

{
  "counter_number": 1
}

✅ Example Response (200 OK)

{
  "message": "เรียกคิวสำเร็จ",
  "data": {
    "id": 1,
    "customer_name": "สมชาย",
    "queue_number": 1,
    "status": "called",
    "counter_number": 1,
    "created_at": "2023-10-25T10:00:00.000Z"
  }
}
GET /queues/active ดึงข้อมูลคิวที่กำลังเรียก (สำหรับหน้าจอ TV)

📝 รายละเอียด

ดึงรายการคิวล่าสุด 5 คิว ที่กำลังถูกเรียกไปที่ช่องบริการต่างๆ เพื่อนำไปแสดงผลบนหน้าจอ Display

✅ Example Response (200 OK)

[
  {
    "queue_number": 1,
    "counter_number": 1
  },
  {
    "queue_number": 2,
    "counter_number": 2
  }
]
GET /queues/waiting ดึงรายการคิวที่รออยู่ทั้งหมด

📝 รายละเอียด

ดึงรายการคิวที่มีสถานะเป็น 'waiting' เรียงตามลำดับหมายเลขคิว

✅ Example Response (200 OK)

[
  {
    "id": 3,
    "customer_name": "สมหญิง",
    "queue_number": 3,
    "status": "waiting",
    "counter_number": null,
    "created_at": "2023-10-25T10:05:00.000Z"
  }
]
GET /reserve/:id ค้นหาคิวด้วย ID

📝 รายละเอียด

ดึงข้อมูลรายละเอียดของคิวที่ต้องการ โดยระบุ id ต่อท้าย URL

✅ Example Response (200 OK)

{
  "id": 1,
  "customer_name": "สมชาย",
  "queue_number": 1,
  "status": "called",
  "counter_number": 1,
  "created_at": "2023-10-25T10:00:00.000Z"
}
